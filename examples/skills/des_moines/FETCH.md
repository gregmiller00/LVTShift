# Des Moines, IA — Notebook Reference Spec

This skill file documents the data sources, rate constants, and class-mapping
decisions used by `cities/des_moines/model.ipynb`. Read this before editing the
notebook or rerunning the overlay script.

## Notebook + supporting script

- `cities/des_moines/model.ipynb` — main LVT shift notebook. Generated by `scripts/build_des_moines_notebook.py`; do not hand-edit cells if a rebuild is planned.
- `scripts/des_moines_district_overlay.py` — generates the per-parcel tax-district sidecar parquet (school × SSMID × urban-sanitary × TIF overlays + Iowa DoM consolidated rates).
- `scripts/build_des_moines_notebook.py` — regenerates the notebook from source.

## Inputs the notebook reads

```text
cities/des_moines/data/des_moines_mapping_ready_<date>.parquet
cities/des_moines/data/des_moines_districts_<date>.parquet
```

The first comes from the geovizwiz-sb side (`data/jurisidictions/run_des_moines.py`)
and is then deposited into this repo. The second is produced locally by the
overlay script above.

## Parcel source (ETL upstream, not this repo)

Parcels originate in `geovizwiz-sb`:

```text
geovizwiz-sb/data/jurisidictions/run_des_moines.py
  → polygon source: Polk County GIS Auditor parcels FeatureServer
  → assessor source: Polk County Assessor CGI form (CSV scrape, prefix-partitioned A-Z + 0-9)
  → output: data/jurisidictions/data/des_moines/des-moines-ia-parcels.parquet
```

The output is then copied to `LVTShift/cities/des_moines/data/des_moines_mapping_ready_<date>.parquet`
for use here. Re-fetch from the geoviz repo when assessor values refresh (annually,
around assessment-roll publication).

## Polk County ArcGIS layers (Auditor_Export FeatureServer)

```text
BASE: https://gis4.polkcountyiowa.gov/server/rest/services/Auditor/Auditor_Export/FeatureServer
```

Layers consumed by the overlay script:

| Layer ID | Name | Used for |
|---|---|---|
| 75 | `School_Districts` | school district name per parcel |
| 80 | `SSMID_Districts` | downtown / neighborhood SSMID surcharges |
| 72 | `Sanitary_Sewer_Districts` | URB WIND-HTS SS and URB SS modifiers |
| 141 | `TIF_Districts` | TIF-captured flag + TIF district name |
| 5 | `Cities` | (optional sanity check) — DM vs W Des Moines |

The overlay script fetches each, spatially joins via representative-point-within
against parcels, then maps the result to the abbreviated names used in the
FY26 consolidated rate PDF. Name lookups are in the script's top constants;
update them if Polk renames a district.

## FY26 rate sources

**Iowa rollback (Assessment Limitation) — AY 2024 / FY 2026:**
- Residential: **47.4316%** (also applies to multi-residential, which folded into residential in 2022)
- Commercial / Industrial / Railroad: two-tier per 2022 Iowa Acts ch 1061. First $150,000 of value uses the residential rate (47.4316%); remainder uses 90.0%.
- Agricultural: productivity-based; FY26 placeholder used in the notebook is `0.8907` (verify against the Iowa DoR Assessment Limitation Order for AY 2024).
- Utility: 100% (no rollback).

Source: [Iowa Legislative Services Agency — Assessment Limitations / Property Value Rollbacks](https://www.legis.iowa.gov/docs/publications/FTNO/1543666.pdf)

**Polk consolidated tax levies — FY 2025-26 (2024 Assessed Valuations - Taxes Payable Sep 2025 and March 2026):**
- Source: Polk County Auditor, [Tax Rate and Valuation Information](https://www.polkcountyiowa.gov/county-auditor/property-tax/tax-rate-and-valuation-information/) → "Polk Consolidated/Net TIF Rates" PDF.
- The 17 DM-specific tax districts (non-AG) and their consolidated millage rates are hardcoded in `des_moines_district_overlay.py` (`DM_TAX_DISTRICTS_FY26`). Mean across DM parcels: 42.43 mills.

Annual maintenance: when Polk publishes the FY 2026-27 PDF (around July 2026),
re-parse it and update both the dict in the overlay script and the rollback
constants in the notebook builder. Then rerun:

```bash
python scripts/des_moines_district_overlay.py    # refreshes the sidecar
python scripts/build_des_moines_notebook.py      # regenerates the notebook
```

## Polk class → rollback bucket

Polk's `property_land_use_category` (Polk Assessor `Class` field) is mapped to
five rollback buckets in the notebook. Distribution observed on 2026-05-23
(`des_moines_mapping_ready_2026_05_22.parquet`):

| Bucket | Parcels | Class strings |
|---|---|---|
| residential   | 43,413 | Residential, Res 3+, Comm-Res3+, Condo Header, Other |
| commercial    | 2,544  | Comm, Industrial |
| exempt        | 1,534  | Govt, School, * Exempt suffixes |
| agricultural  | 110    | Agricultural, Ag-Ag Res |
| utility       | 33     | Utility |

Exempt parcels are excluded from the revenue-neutral solve.

## Methodology choices (recorded per LVT_MODELING_GUIDE §Q1.0)

- **Policy scope:** full levy stack (city + county + school + community college + assessor + ag-ext + state).
- **Reform shape:** split-rate at 2:1, 4:1, 8:1 + 100% building abatement.
- **District granularity:** multi-overlay (school × SSMID × urban-sanitary). Each parcel's millage reflects its actual district combo.
- **TIF treatment:** dual reporting. All scenarios run twice — once including TIF-captured parcels, once excluding (~34% of taxable revenue sits inside TIF).
- **Current structure preserved:** yes. Rollback, post-tax credits, and exemptions all remain unchanged by the reform.

## Known limitations / verify-before-publishing

1. **Agricultural rollback** is a placeholder constant — replace with the precise AY 2024 value from the Iowa DoR Assessment Limitation Order.
2. **TIF mechanics** simplified to "TIF parcels contribute zero to general fund." Real Iowa TIF math (frozen base + increment capture + residual) is more granular; the simplification overstates for newer TIF districts.
3. **Credits** (homestead, military, business property tax credit replacement) not modeled. Per the modeling guide, they're preserved unchanged by the reform and don't affect the revenue-neutral solve on taxable values.
4. **No per-parcel actual-bill validation.** Compare modeled `current_tax` to Polk's published per-parcel bills on a sample of ~500 parcels before publishing any results.
